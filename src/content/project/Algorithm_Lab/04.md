#### Question1: Try to proof that minimum spanning tree of $G$ ...  
Ans:  
1. Since $T$ spans all vertices, there is a unique path in $T$ from $u$ to $v$. Adding edge $e$ to $T$ creates a cycle.
2. This cycle must contain some other edge $e'=(x,y)$ that also crosses the cut $(C,V∖C)$, because $u∈C$, $v∉C$, and the only way to connect them is via some edge between these two sets.
3. Since $e$ was defined as the minimum-weight edge across this cut, we have: $$w(e)<w(e')$$
4. Now consider a new tree $T'=T−e'+e$. That is, remove $e'$ and add $e$ to break the cycle and preserve the tree.
5. The new total weight is: $$w(T')=w(T)−w(e' )+w(e)<w(T)$$
This contradicts the assumption that $T$ is a minimum spanning tree.

Conclusion:
Therefore, our assumption must be wrong, because every MST must contain all safe edges, as replacing a non safe edge with a safe edge reduces the total weight.
$$QED$$

#### Question2: Please modified above algorithm to letfind minimum spanning forest ...  
Ans:  
``` c++
#include <vector>
#include <list>
#include <queue>

typedef int WeightType;
struct Edge {
    int u, v;
    WeightType w;
};

struct Graph {
    Graph(int n): E(n) {}
    std::vector<std::list<Edge>> E;
    void add_edge(int u, int v, WeightType w) {
        E[u].push_back({u, v, w});
        E[v].push_back({v, u, w});
    }
    int n() const { return E.size(); }
    int m() const {
        int res = 0;
        for (auto& list : E)
            res += list.size();
        return res / 2;
    }
};

std::pair<Graph, int> MSF(const Graph &G) {
    auto WeightComp = [](const Edge &e1, const Edge &e2) {
        return e1.w > e2.w;
    };
    std::priority_queue<Edge, std::vector<Edge>, decltype(WeightComp)> candidates(WeightComp);

    Graph forest(G.n());
    std::vector<bool> visited(G.n(), false);
    int tree_count = 0;

    for (int start = 0; start < G.n(); start++) {
        if (visited[start]) continue;

        // Begin new tree from unvisited node
        visited[start] = true;
        tree_count++;

        for (Edge e : G.E[start])
            candidates.push(e);

        while (!candidates.empty()) {
            while (!candidates.empty()) {
                Edge temp = candidates.top();
                if (visited[temp.u] && visited[temp.v])
                    candidates.pop();
                else
                    break;
            }
            if (candidates.empty())
                break;

            Edge safe = candidates.top();
            candidates.pop();
            int u = visited[safe.v]? safe.u : safe.v;

            forest.add_edge(safe.u, safe.v, safe.w);
            visited[u] = true;

            for (Edge e : G.E[u]) {
                if (!visited[e.v])
                    candidates.push(e);
            }
        }
    }

    return {forest, tree_count};
}
```

#### Question3: Please design a function to detect an edge is useless or not...  
Ans:
``` c++
struct DSU {
    std::vector<int> parent, rank;

    DSU(int n): parent(n), rank(n, 0) {
        for (int i = 0; i < n; ++i)
            parent[i] = i;
    }

    int find(int x) {
        if (x != parent[x])
            parent[x] = find(parent[x]); 
        return parent[x];
    }

    bool unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry)
            return false;
        if (rank[rx] < rank[ry])
            parent[rx] = ry;
        else if (rank[rx] > rank[ry])
            parent[ry] = rx;
        else {
            parent[ry] = rx;
            rank[rx]++;
        }
        return true;
    }
};
```
``` c++
Graph Kruskal(const Graph &G) {
    std::vector<Edge> edges;
    for (int u = 0; u < G.n(); ++u)
        for (auto e : G.E[u])
            if (u < e.v)
                edges.push_back(e);

    std::sort(edges.begin(), edges.end(), [](const Edge &a, const Edge &b) {
        return a.w < b.w;
    });

    DSU dsu(G.n());
    Graph T(G.n());

    for (Edge e : edges) {
        if (dsu.unite(e.u, e.v)) {
            T.add_edge(e.u, e.v, e.w);
        }
    }
    return T;
}
```
1. Let n be the number of vertices, m be the number of edges
2. Sort edges: $O(mlogm)$
3. Process each edge:
* Each find/unite is nearly constant time:  $O(α(n))$, where α is the inverse Ackermann function (very slow-growing).
* Total cost for processing edges: $O(mα(n))$
4. Total time complexity: $$O(mlogm)$$ (dominant term is sorting)

#### Question4: `WeightComp` needs $D_{s,t}$ to measure which edge should choose. Giving...  
Ans:
``` c++
#include <vector>
#include <list>

typedef int WeightType;
struct Edge {
    int u, v;
    WeightType w;
};

struct Graph {
    std::vector<std::list<Edge>> E;
    Graph(int n): E(n) {}
    void add_edge(int u, int v, WeightType w) {
        E[u].push_back({u, v, w});
        E[v].push_back({v, u, w});
    }
};

void DFS(const Graph &T, int root, std::vector<WeightType> &dist, std::vector<bool> &visited, WeightType acc = 0) {
    visited[root] = true;
    dist[root] = acc;
    for (auto e : T.E[root]) {
        if (!visited[e.v]) {
            DFS(T, e.v, dist, visited, acc + e.w);
        }
    }
}

// usage
// std::vector<WeightType> dist(n, 0);
// std::vector<bool> visited(n, false);
// DFS(tree, root, dist, visited);
// Then dist[v] contains the cost from root to vertex v
```
1. We visit every node once and process every edge once.
2. In a tree with $n$ nodes, there are $n−1$ edges. 
3. Time Complexity = $O(n)$

#### Question5: Brief what is A* algorithm and explain why A* can ...  
Ans:  
1. A pathfinding algorithm is used to find the lowest-cost path between nodes in a graph, particularly effective in 2D or 3D spatial environments. This algorithm combines the strengths of Greedy Best-First Search and Dijkstra’s Algorithm: it achieves faster performance through heuristic-guided search while ensuring that the found path is optimal, provided the heuristic function meets certain conditions (such as consistency or monotonicity). The algorithm uses an evaluation function: $f(n)=g(n)+h(n)$
2. A* is guaranteed to find the optimal (shortest) path if the heuristic function $h(v)$ satisfies the following condition:
* Admissibility:
The heuristic never overestimates the actual minimum cost to reach the goal: $$h(v)≤true cost from v to goal$$
* (Optional) Consistency (Monotonicity):
A stronger condition where: $$h(v)≤w(v,u)+h(u)$$
for every edge $(v,u)$.
This ensures $f(v)$ is non-decreasing along a path.

If the heuristic is admissible, A* will always expand the optimal path first and stop as soon as it reaches the goal. Thus, it finds the shortest path.
