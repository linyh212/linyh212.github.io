## Question1: Rewrite the above algorithm to find 3rd-largest element.  
### Ans:  
```c++
std::vector<DataType> solve(DataSet input) {
    // If the input is small, just return the sorted top 3
    if (input.count <= 3) {
        std::vector<DataType> result;
        for (size_t i = 0; i < input.count; i++)
            result.push_back(given[input.begin + i]);
        std::sort(result.begin(), result.end(), std::greater<DataType>());
        return result;
    }

    // Divide problem into 2 subproblems
    std::vector<DataSet> subProblems(2);
    subProblems[0].begin = input.begin;
    subProblems[0].count = input.count / 2;
    subProblems[1].begin = input.begin + subProblems[0].count;
    subProblems[1].count = input.count - subProblems[0].count;

    // Recursively solve each subproblem
    std::vector<DataType> leftTop = solve(subProblems[0]);
    std::vector<DataType> rightTop = solve(subProblems[1]);

    // Merge top values from both subproblems
    std::vector<DataType> combined = leftTop;
    combined.insert(combined.end(), rightTop.begin(), rightTop.end());

    // Sort combined and return top 3
    std::sort(combined.begin(), combined.end(), std::greater<DataType>());
    if (combined.size() > 3)
        combined.resize(3);

    return combined;
}

// To get the 3rd largest:
DataType getThirdLargest() {
    DataSet full = {0, given.size()};
    std::vector<DataType> top3 = solve(full);
    if (top3.size() < 3)
        throw std::runtime_error("Not enough elements to find 3rd largest.");
    return top3[2];
}
```
## Question2: When satisfied otherwise, try to proof ...  
### Ans:  
1. Let $f(n) = \Theta(n^{c_{\text{crit}}} \log^k n)$ for some constant k>=0, where $c_{\text{crit}} = \log_b a$. Then, for any constant $c > c_{\text{crit}}$, it holds that $f(n) = o(n^c)$. That is, $$\lim_{n \to \infty}\frac{f(n)}{n^c} = 0$$  
Thus, $f(n) \notin \Omega(n^c)$ for any $c > c_{\text{crit}}$.

2. Assume: $f(n) = \Theta(n^{c_{\text{crit}}} \log^k n)$  
Then, by the definition of $\Theta$-notation, there exist constants $C_1, C_2 > 0$ and $n_0 \in \mathbb{N}$ such that for all $n \geq n_0$,  
$$C_1 n^{c_{\text{crit}}} \log^k n \leq f(n) \leq C_2 n^{c_{\text{crit}}} \log^k n$$
Let $c > c_{\text{crit}}$. Then, consider the ratio: $$\frac{f(n)}{n^c} = \Theta\left(\frac{n^{c_{\text{crit}}} \log^k n}{n^c} \right) =\Theta\left( n^{c_{\text{crit}} - c} \log^k n \right)$$
Since $c - c_{\text{crit}} > 0$, we have $c_{\text{crit}} - c < 0$, and therefore: $$n^{c_{\text{crit}} - c} \log^k n = \frac{\log^k n}{n^{c - c_{\text{crit}}}}$$
As $n \to \infty$, $n^{c - c_{\text{crit}}} \to \infty$ faster than any polylogarithmic function $\log^k n$, so: $$\frac{\log^k n}{n^{c - c_{\text{crit}}}} \to 0$$
Hence, $$\displaystyle\lim_{n \to \infty} \frac{f(n)}{n^c} = 0\Rightarrow f(n) = o(n^c) \Rightarrow f(n) \notin \Omega(n^c)$$  

3. If $f(n) = \Theta(n^{c_{\text{crit}}} \log^k n)$, then for any $c > c_{crit}$, it follows that $f(n) = o(n^c)$, and thus $f(n) \notin \Omega(n^c)$.  
$$\boxed{Q.E.D.}$$  

## Question3: Try to analyse the time complexity ...  
### Ans:  
When k = 5 using the Master Theorem, the recurrence relation becomes: $$T(n) = 5T\left(\frac{n}{5}\right) + cn$$

This means the problem is divided into 5 subproblems of size $\frac{n}{5}$, and the merging step takes linear time, $cn$. Applying the Master Theorem in the form: $$T(n) = aT\left(\frac{n}{b}\right) + f(n)$$

we have: $a = 5$, $b = 5$, $f(n) = \Theta(n)$.

We compute: $$n^{\log_b a} = n^{\log_5 5} = n^1$$

Since $f(n) = \Theta(n) = \Theta(n^{\log_b a})$, this falls under **Case 2** of the Master Theorem. Therefore, the time complexity is: $$T(n) = \Theta(n \log n)$$

In conclusion, even if merge sort divides the array into 5 parts instead of 2, the overall time complexity remains $\Theta(n \log n)$, assuming the merge step takes linear time.

## Question4: Suppose given elements are not unique, please modify ...
### Ans: 
``` c++
#include <stdlib.h>
using namespace std;

vector<pair<int, int>> mergeCount(const vector<pair<int, int>>& left, const vector<pair<int, int>>& right) {
    vector<pair<int, int>> merged;
    int i = 0, j = 0;
    while (i < left.size() && j < right.size()) {
        if (left[i].first < right[j].first) {
            merged.push_back(left[i++]);
        }
        else if (left[i].first > right[j].first) {
            merged.push_back(right[j++]);
        }
        else {
            merged.emplace_back(left[i].first, left[i].second + right[j].second);
            ++i;
            ++j;
        }
    }
    while (i < left.size()) merged.push_back(left[i++]);
    while (j < right.size()) merged.push_back(right[j++]);
    return merged;
}
vector<pair<int, int>> mergeSortCount(const vector<int>& arr, int l, int r) {
    if (l > r) return {};
    if (l == r) return {{arr[l], 1}};
    int mid = l + (r - l) / 2;
    auto left = mergeSortCount(arr, l, mid);
    auto right = mergeSortCount(arr, mid + 1, r);
    return mergeCount(left, right);
}

int main() {
    vector<int> input;
    string line;
    getline(cin, line);
    stringstream ss(line);
    int num;
    while (ss >> num) {
        input.push_back(num);
    }
    auto result = mergeSortCount(input, 0, input.size() - 1);
    for (const auto& [value, freq] : result) {
        cout << "{" << value << ", " << freq << "} ";
    }
    cout << endl;
    return 0;
}
```
