# Note #06 10/07/25  

# 會議主題: YOLO 模型初步訓練與結果分析  

## 一、會議目的  
- 紀錄 YOLO 模型初步訓練進度與結果  
- 分析訓練過程中的挑戰與問題  
- 討論後續模型優化與資料擴增方向  

## 二、討論內容與決議  

### 1. 當前進度  
- 已完成 YOLOv11n 初步訓練  
- 模型能在影片中辨識出 paddle，但準確率仍受資料量限制  
- 訓練集大小：約 304 張（經增強後擴增三倍）  
- 訓練週期：100 epochs，影像尺寸：640×640  
- 現階段使用 **MacBook M2 CPU** 進行訓練，訓練效率有限  
- 未來將改成**MacBook M2 CPU**，若有機會可使用 **RTX 3090 / 4090 GPU**，將能大幅提升訓練速度與模型表現  

### 2. 問題與挑戰  
- **環境雜訊與誤檢**：背景中金屬器材與 paddle 外型相似，可能同時被框出  
- **標註精度不足**：部分框標註過寬，導致模型邊界學習不精準  
- **資料量不足**：角度、光源條件偏少，模型在新情境下容易誤判  

### 3. 訓練結果  
- **box_loss / cls_loss / dfl_loss**：訓練與驗證曲線平滑下降，無過擬合  
- **precision / recall**：接近 0.9，能正確識別大部分 paddle  
- **mAP50-95**：趨近 0.85，模型在多 IoU 門檻下仍具良好準確度  
- 整體訓練過程穩定收斂，顯示模型具備良好學習能力  

### 4. 未來工作決議  
- 建立更完整 Ground Truth，擴充標註資料集  
- 收集多角度、不同光源的影像樣本，提升泛化能力  
- 嘗試調整 NMS 門檻與信心值，降低誤檢  
- 視資料情況考慮升級至 YOLOv11s 或 anchor-free 架構  

## 三、未來修正與補充工作  
1. **建立 Ground Truth**  
   - 評估模型誤檢情形，擴充標註資料集  
   - 建立多角度、不同光源的真實標註影像以強化泛化能力  
2. **模型微調**  
   - 嘗試調整 NMS 門檻與信心值，減少同時框到多物件的情況  
   - 若資料充足，考慮升級模型至 YOLOv11s 或使用自定義 anchor-free 結構  
3. **後處理分析**  
   - 針對 paddle 框的幾何形狀設篩選條件（如 aspect ratio）以過濾異常預測框  

## 四、後續思考  
- 在 CPU 訓練環境下，如何有效控制模型大小與訓練時間  
- 若取得高階 GPU（RTX 3090/4090），可進行更長時間與更大模型的實驗  
- 逐步建立標準化資料處理與標註流程，確保數據品質
