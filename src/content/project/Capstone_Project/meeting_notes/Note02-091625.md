# Note #02 09/16/25  

# 會議主題： 討論 YOLO 特徵提取與 GRU 動作辨識流程  

## 一、 會議目的  
本次會議旨在確認 YOLO 偵測結果如何轉換為可供 GRU 模型使用的時序特徵向量，並檢視現有流程圖與技術文檔的不足之處，後續將與教授討論詳細事宜。

## 二、 討論內容與決議  

### 1. 特徵提取與預處理（初版流程）  

**輸入**：YOLO 偵測輸出（bbox_x, bbox_y, bbox_width, bbox_height, confidence, class）  

**處理步驟**：  
1. **數據分組 (Grouping)**  
   - 槳與手歸為同一組，作為控制器組合。  
2. **空間特徵計算 (Spatial Features)**  
   - 相對位置：Δx, Δy  
   - 距離：標準化歐式距離  
   - 角度：選用，計算與水平線夾角  
   - 寬高比：bbox_width / bbox_height  
3. **歸一化 (Normalization)**  
   - 特徵縮放至 [-1, 1] 或 [0, 1]  
4. **構建特徵向量 (Feature Vector)**  
   - 例如 [Δx, Δy, distance, aspect_ratio_A, confidence_A, aspect_ratio_B, confidence_B, …]  
  
**輸出**：單幀特徵向量（假設 8 維）。  

---

### 2. GRU 模型訓練與推論  

1. **數據收集與標註**  
   - 錄製操作影片 → YOLO 偵測 → 特徵向量 → 組成時序序列  
   - 每段序列（如 10 幀）標註為特定動作（如 “moving_left”）  
2. **模型訓練**  
   - 輸入：形如 [10, 8] 的序列（10 時間步 × 8 特徵）  
   - 輸出：對應動作標籤  
   - 需大量標註數據才能學習到特徵變化 → 動作關聯  
3. **使用（推論）**  
   - 輸入影片 → 輸出動作標籤（如 “moving_left”）  

---

### 3. 當前不足與需改進之處
- 流程圖尚未完整，僅為草稿，需重新繪製完整架構  
- YOLO 偵測「船槳」準確率不穩定，存在無法正確鎖定情況  
  → 需研究改善方案（如加標記、資料增強、後處理演算法）
- 船槳偵測問題：
  - 目前多數情況只能斜對角框出兩端點，未必能完整覆蓋槳輪廓、角度  
  - 後續需探討是否可針對黑色船槳強化偵測
