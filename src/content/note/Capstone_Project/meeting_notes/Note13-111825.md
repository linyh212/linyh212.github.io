# Note #13 11/18/25  

# 會議主題: 模型測試異常結果與問題排查  

## 一、會議目的  
- 紀錄 YOLO 模型在外部影片測試時出現的問題  
- 初步判斷可能的訓練資料錯誤來源  
- 規劃改善資料集與驗證流程  

---

## 二、問題描述  
在將網路上找到的「划船機影片」輸入目前的 YOLO 模型進行測試時，出現以下現象：  

- 模型完全沒有框住船槳  
- 預測框反而落在畫面邊緣或背景位置  
- 判斷極可能為模型偵測錯誤目標，非真正的船槳  

---

## 三、初步推測原因  

### 1. 訓練資料標註方式可能錯誤  
- 目前資料內容是「已經包含框住船槳的照片」  
- 若標註框是根據這些框（或誤標），模型可能學到的是『框的形狀或位置』而不是『船槳本體』  
- 因此推測模型正在「尋找框框」，而不是「尋找船槳」  

### 2. 資料集缺乏真實背景多樣性  
- 訓練資料主要為固定環境、固定角度  
- 外部影片背景更複雜 → 模型無法辨識  

### 3. 訓練集與驗證集未能有效偵測資料問題  
- 過去訓練中，test set 未被明確使用  
- 可能導致訓練資料錯誤無法被及早發現  

---

## 四、決議與改善方向  

### 1. 重新建立資料集比例  
採用 **YOLO 常用比例 8:1:1**  
- 8 → train  
- 1 → val  
- 1 → test（新加入）  

新增 **嚴格 test set** 用來：  

- 驗證模型是否真的學到「船槳」特徵  
- 明確區分訓練資料錯誤 vs 背景雜訊造成誤判  

### 2. 重新檢查標註流程  
- 檢查是否「把含框照片」直接當成原始影像  
- 若有誤標、偽標 → 必須重新標記  
- 若框的位置過於一致 → 高機率造成模型過度依賴背景  

### 3. 增加背景多樣性  
- 蒐集更多不同場景、不同角度的划船影像  
- 確保模型不依賴單一場景特徵  

### 4. 針對新 test set 優先測試  
- 若新 test 結果正常 → 原本外部影片只是 domain gap  
- 若新 test 仍異常 → 資料與標註流程必須全面重建  

---

## 五、後續工作  
- 本週開始重新製作資料集（8:1:1 分配）  
- 建立新 test pipeline 以快速檢查資料錯誤  
- 下週預計重新訓練 YOLO 模型並再次測試網路影片
