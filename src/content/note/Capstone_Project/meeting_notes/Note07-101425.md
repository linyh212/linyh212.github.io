# Note #07 10/14/25  

# 會議主題: 專案進度報告 — 划船動作偵測資料標記進展  

## 一、會議目的  
- 確認目前槳（paddle）標記進度與品質  
- 討論半自動標記工具的效能與修正流程  
- 為後續 YOLO 模型訓練準備乾淨且一致的資料集  

---

## 二、討論內容與決議  

### 1. 標記流程與工具  

**半自動標記（Roboflow RF-DETR 模型）**  
- 採用 Roboflow 平台內建的 **RF-DETR (Region-based DETR)** 模型進行半自動標記。  
- 模型可自動預測影像中槳的位置，顯著減少人工框選時間。  
- 對於 **傾斜角度大** 或 **背景顏色相近** 的影像，RF-DETR 的預測仍需人工後處理。  

**手動微調**  
- 對部分偏移或框選不準的標記進行人工修正。  
- 修正重點：  
  - 槳頭邊緣模糊或被衣物遮擋的情況。  
  - 槳柄與背景顏色重疊的部分。  
- 經人工檢查後，標籤框與真實物件對齊度達 **95% 以上**。  

---

### 2. 資料輸入與模型訓練準備  

**標記資料**  
- 影片擷取影像數：約 **40,000 張**  
- 影像增強後總數：約 **120,000 張**  
- 類別數：**1（paddle）**  
- 格式：**YOLOv11 標準格式（.txt 標籤檔 + .jpg 影像）**  

**模型與硬體資源**  
- 計畫使用 **YOLOv8** 模型進行訓練。  
- 預計使用具 **GPU 加速（如 RTX 3090）** 的硬體以縮短訓練時間。  
- 訓練環境：  
  - Python 3.13.7  
  - PyTorch  
  - Ultralytics  
  - Roboflow SDK
