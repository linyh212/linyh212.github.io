# Note #27 03/10/26  

# 會議主題： Body Pose 偵測優化與 3D 轉換進度  

## 一、會議目的
- 更新目前人體姿態偵測（Body Pose）進度
- 討論多人物偵測時的篩選條件
- 確認 2D → 3D 模型訓練進度
- 檢視網頁 UI 改進方向

---

## 二、目前進度

### 1. Body Pose 偵測調整
目前人體偵測在多人物畫面中需要加入額外篩選條件，以確保只追蹤主要目標人物。

新增條件如下：
- **最靠近畫面中心點的人物**
- **Bounding box 面積需大於一定數值**

目的：
- 避免遠距離人物或背景人物被誤選為主要追蹤對象
- 提升骨架 keypoints 生成的穩定性

---

### 2. 左右腳辨識問題
目前在部分情況下會出現 **左右腳辨識混淆** 的問題，例如：
- left leg / right leg 對調
- 部分 frame 偵測不穩定

評估結果：
- 此問題 **不影響最終需要計算的數據**
- 因最終分析主要依賴：
  - 關節角度變化
  - 動作週期
  - 運動趨勢

若後續影響數據分析，再進一步調整模型或後處理。

---

### 3. 2D → 3D 轉換模型
目前正在進行 **2D skeleton 轉 3D skeleton** 的訓練。

進度目標：
- **希望本週完成模型訓練**
- 完成後將測試：
  - 3D 骨架生成穩定度
  - 動作角度計算是否正常

---

### 4. 網頁 UI 進度
目前 UI 功能已可基本運作，接下來將進行 **介面美化與配置優化**。

可能優化方向：
- 頁面排版與視覺設計
- 結果顯示區域重新配置
- 影片播放與3d影像同步顯示

後續可思考：
- **分析結果頁面如何配置**
- 是否需要：
  - 骨架視覺化區域
  - 數據圖表區
  - 影片控制區

---
